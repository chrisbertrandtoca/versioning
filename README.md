# 📦 Project: The Versioning Vortex 🌀
### *“It worked on my machine, and in v1.0.2-alpha.final-FIXED-DEAR-GOD-PLEASE”*

Welcome to the official repository for the **Versioning Demo**. This project is a living testament to the fact that naming things is hard, and tracking changes is harder. Whether you’re a SemVer purist or a "just-push-to-main" anarchist, there’s something here for you to regret.

---

## 🛠 Features
* **Arbitrary Numbering:** Experience the thrill of jumping from `v1.0.0` to `v2.0.0` just because we changed a button from #F0F to #F1F.
* **Dependency Hell Simulation:** A real-time demo of what happens when a package you've never heard of gets unpublished.
* **The "Legacy" Folder:** A digital graveyard of files named `script_v2_final_FINAL_v3.js`.
* **Git Blame Integration:** Automatically highlights the person who left the Zoom call first.

---

## 🚦 Getting Started

To get this demo running, please follow these steps (assuming you have the patience of a saint):

1.  **Clone the repo:**
    `git clone https://github.com/YourUsername/versioning-nightmare.git`
2.  **Install dependencies:**
    `npm install` *(Go grab a coffee. Maybe a three-course meal.)*
3.  **Run the demo:**
    `npm start`

## 🔢 Our Versioning Philosophy
We strictly adhere to **Emotional Versioning (EmoVer)**:

| Version | Meaning |
| :--- | :--- |
| **0.0.1** | "It compiles, but don't look at it." |
| **1.0.0** | "I'm confident enough to lie to the client." |
| **1.1.0** | "Added a feature; accidentally broke the login page." |
| **2.0.0** | "I deleted the old CSS and now I'm scared." |
| **9.9.9** | "The intern pushed directly to production on a Friday." |
